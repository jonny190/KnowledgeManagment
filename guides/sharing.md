# Sharing notes

Notes can be shared with specific people, made visible to everyone in your workspace, or published via a public link that anyone can open without signing in.

## Opening the Share dialog

Open any note and click the Share button in the top-right corner of the note header. The dialog shows two sections: the list of people who currently have access, and the public link panel.

If the note has already been shared with someone or has an active public link, a small Shared badge appears next to the Share button so you can tell at a glance.

## Adding someone by email

In the Share dialog, type the email address of the person you want to share with. Choose a role from the dropdown next to the input field:

- View: the person can read the note but cannot edit it.
- Edit: the person can read and edit the note in the collaborative editor.

Click the Share button. If no account exists for that email address the dialog will tell you. Otherwise the person appears immediately in the list of people with access.

## Changing or removing access

Each person in the access list has a role dropdown and a Remove button. Change the dropdown to swap between View and Edit. Click Remove to revoke access entirely.

## Controlling who in your workspace can see the note

For notes in a workspace vault the Share dialog includes a Visibility section. Two options are available:

- Everyone in workspace: all workspace members can read and edit the note. This is the default for new notes in a workspace vault.
- Only people I share with: the note becomes private. Workspace members who are not listed in the access list cannot open it.

Personal vault notes are always private and do not show the Visibility section.

## Creating a public link

In the Public link section of the Share dialog, click Create public link. A link is generated and shown as a URL you can copy to your clipboard. Anyone who has the link can open the note in a read-only view without signing in.

A note can have multiple public links, but the dialog surfaces one at a time. Creating a new link while one already exists adds a second link; both remain active until revoked.

## Revoking a public link

Click Revoke next to the link in the Share dialog. The link stops working immediately. Anyone who tries to open the old URL will see a Not found page. Any other active links for the same note continue to work until they are individually revoked.

## What a shared user sees

A person with Edit access can open the note in the full collaborative editor, see other connected users in the active-users bar, and make changes that sync to everyone else in real time.

A person with View access can read the note but cannot type in the editor. The realtime connection requires Edit permission, so the note is shown as a static read-only page rather than the live editor.

A person who opens a public link without signing in sees the same read-only static page and has no access to any other part of the workspace.
